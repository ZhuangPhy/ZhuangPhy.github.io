小时候有个邻居大爷，各种形状的木头在他的工具底下，接口处的精准预设，形成了各种家具。那一排不同的锯子，斧头，锤子等闪闪发亮。还有父亲那十几个柜子的电子原件，是怎么维护着不同家庭电器的运转。小小的我感到如此的炫酷：这是属于创造的浪漫，来自于改造这个世界的不安分。

后来年少时阅读人类的演化史，明白了人类弱小的身躯能在大自然的丛林中纵横，来源于对工具的使用和人类的合作能力。拿起厚重而锋利的冲锋锯的时候，你的灵魂仿佛与百万年前那个举起木棒的先祖相连。

马克思认为：“通过参与自然和改变自然的能力，个体才能表现出人类物种独有的高级意识。”因为恐惧而改造，因为欲望而创造。这，就是人。这，就是工程师。

作为新时代的人，本质没有改变，但是有了更丰富的表象。我们有计算机，互联网，物联网，更好的摄影器材。所以工程师文化的概念随着变广。对，说这么多只是因为我不喜欢“肥宅”这个词，我是在家里也可以改造世界，我更喜欢说“技术宅”。

2 电子技术

GPU

最早对GPU的认识就是一个用来处理图形的处理器。在16年掀起的机器学习浪潮中，知道了GPU在机器学习领域拥有很强的优势。

曾经，IBM为了模拟油气勘探，不得不动用70万个处理器、近半个足球场的空间。而用GPU加速后，仅需两台服务器和半张乒乓球桌的大小就全部搞定。那么在哪些领域的程序适合在GPU上运行。

  • 计算密集型(Compute-intensive)的程序。就是其大部分运行时间花在了寄存器运算上，寄存器的速度和处理器的速度相当，从寄存器读写数据几乎没有延时。

  • 易于并行的程序。GPU其实是一种SIMD(Single Instruction Multiple Data)架构， 他有成百上千个核，每一个核在同一时间最好能做同样的事情。

我电脑里运行manim的过程记录

打开软件Cygwin Terminal的界面，通过以下指令进入到对应文件夹

cd d:
cd program
cd manim-master

manim-master 这个文件夹里面有一个python文件 example_scenes.py，用指令运行

python -m manim example_scenes.py WriteStuff -pl

manim-master里面也可以设立文件夹方便分类，比如我设立文件夹physics001，里面有一个python文件Rotate_by_complex.py ，运行这个文件，使用指令

python -m manim physics001/Rotate_by_complex.py WriteStuff -pl

遇到的问题&解决方法

(1)数学公式的渲染冲突

Hexo默认使用hexo-renderer-marked来渲染Markdown，但是这个渲染器在解析Latex语法的时候会和Markdown本身的语法符号产生冲突。网上有很多解决冲突的办法，有的需要手动修改一些js文件，有的需要手动转义。这里我推荐最便捷的方法是直接换一个渲染器Pandoc

npm uninstall hexo-renderer-marked --save
npm install hexo-renderer-pandoc --save

Typora编辑器使用的就是Pandoc的语法。
